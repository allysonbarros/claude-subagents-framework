# Data Engineering

Agentes especializados em pipelines de dados, data warehousing, big data e ferramentas de processamento de dados.

## Propósito

Os agentes de Data Engineering ajudam a:
- Projetar e implementar pipelines de ETL/ELT
- Otimizar data warehouses
- Gerenciar big data e processamento distribuído
- Implementar estratégias de armazenamento de dados
- Orquestrar workflows de dados
- Otimizar performance de queries e índices

## Agentes Disponíveis

### [Databricks Specialist](./databricks-specialist.md)
Especializado em Databricks e Apache Spark. Use para construir pipelines escaláveis, implementar Delta Lake e otimizar processamento distribuído de dados.

### [dbt Specialist](./dbt-specialist.md)
Especializado em dbt (Data Build Tool) para transformação de dados. Use para modelar dados, criar transformações SQL e implementar data quality checks.

### [Snowflake Specialist](./snowflake-specialist.md)
Especializado em Snowflake data warehouse. Use para design de schemas, otimização de queries, implementar time-travel e performance tuning.

### [Spark Specialist](./spark-specialist.md)
Especializado em Apache Spark para processamento distribuído. Use para implementar jobs Spark, otimizar particionamento e trabalhar com RDDs/DataFrames.

## Casos de Uso Comuns

- Construção de data warehouses
- Pipelines de ETL/ELT
- Processamento de dados em larga escala
- Data lake implementation
- Real-time data streaming
- Data quality e governance
- Analytics e BI
- Migração de dados legados

## Combinações Recomendadas

- **Data Engineering + IA/ML**: Para preparação e transformação de dados para modelos
- **Data Engineering + Analytics**: Para infraestrutura de dados que alimenta analytics
- **Data Engineering + Devops**: Para deployment e orquestração de pipelines
- **Data Engineering + Backend**: Para arquitetura de dados em aplicações
